# Phase 1: Function autocomplete + signature hints

## Overview

While editing a formula (leading `=`), typing a function-name prefix opens a completion
dropdown of matching engine-registered function names; accepting one inserts `NAME(` with
the caret placed just after the paren, and a passive one-line signature hint shows the
argument template. Identical behavior in the data-row (formula bar) editor and the in-cell
overlay editor, which share one pending edit.

Per the resolved owner decisions: D1.1 = static whole-template hint (no current-arg
tracking); D1.2 = insert `NAME(` (no trailing `)`), caret after the paren; D1.3 = the 345
engine-registered names; D1.4 = trigger at ≥1 identifier char after `=`.

Chrome-only: no pixel/render suite. The pure catalog + context logic lives in
`freecell-core` (IronCalc-free); all UI state + wiring lives in `freecell-app` (`ChromeView`
owns the state; the grid renders + intercepts keys for the in-cell overlay only).

## Steps

### A. `freecell-core` — static catalog + completion-context logic (new `functions.rs`)

1. New module `app/crates/freecell-core/src/functions.rs`, registered in `lib.rs` as
   `pub mod functions;` (no crate-root re-exports — callers use the fully-qualified
   `functions::complete` / `functions::signature` / `functions::fn_edit_context` /
   `functions::enclosing_fn_name` path, consistent with the other pure modules).
2. `pub struct FnSig { pub name: &'static str, pub template: &'static str, pub rank: u16 }`.
   `pub const FUNCTIONS: &[FnSig]` — the 345 engine-registered names from
   `experiments/round-2/03-function-parity/data/ironcalc_functions.csv`. Each of the 154
   `common`-importance names (per `excel_functions_canonical.csv`) gets `rank = 0` and an
   authored argument template (`"SUMIF(range, criteria, [sum_range])"`, optional args in
   `[]`); the remaining 191 get `rank = 1` and a generic fallback template `"NAME(…)"`
   (the name uppercased + `(…)`). Generated by a committed one-shot script from the two
   CSVs and pasted in as a plain `const` array (no build-time codegen).
3. `pub fn complete(prefix: &str) -> Vec<&'static FnSig>` — case-insensitive prefix match;
   order = exact-name-first, then `rank` ascending, then name ascending. Empty prefix →
   empty (the ≥1-char guard lives in `fn_edit_context`, but `complete("")` is also empty as
   a belt-and-braces).
4. `pub fn signature(name: &str) -> Option<&'static FnSig>` — exact case-insensitive lookup.
5. `pub struct FnEditContext { pub token_start: usize, pub prefix: String }` and
   `pub fn fn_edit_context(text: &str, caret: usize) -> Option<FnEditContext>` — the lexical
   heuristic (byte offsets; identifier chars `[A-Za-z0-9_.]`, leading char a letter):
   - `text` must start with `=`; caret within bounds.
   - Not inside a string literal at `caret` (odd count of unescaped `"`, doubled `""`
     escape) → else None.
   - Walk left from `caret` over identifier bytes to `token_start`; `prefix =
     text[token_start..caret]`; require len ≥ 1 and `prefix` first char is a letter.
   - Function-position gate: the char before `token_start` must be start-of-formula (`=` at
     index 0), or one of `( , + - * / ^ & < > = % :` / whitespace; a digit or `)` → None.
   - Cell-ref guard: if `prefix` has cell-reference shape (`[A-Za-z]{1,3}[0-9]+`, letters
     then digits) **and** no catalog name starts with it → None (so `=A1` is a ref, but
     `=LOG10`/`=ATAN2` — real names with a matching prefix — still complete).
6. `pub fn enclosing_fn_name(text: &str, caret: usize) -> Option<&str>` — scan `text[..caret]`
   (skipping string literals), track a stack of open-paren byte offsets; the innermost still-
   open `(` gives the enclosing call; read the identifier immediately left of it and return
   it (leading char must be a letter). Drives the signature hint's "caret inside a call".

### B. `freecell-app` — `ChromeView` state + wiring (`chrome/view.rs`, `chrome/mod.rs`)

7. New display types in `chrome/mod.rs`: `AutocompleteDisplay { rows: Vec<AutocompleteRow>,
   highlight: usize }` and `AutocompleteRow { name: SharedString, template: SharedString }`
   (grid-renderable, `Debug + Clone`). Extend `ChromeGridRequest::EditState` with
   `autocomplete: Option<AutocompleteDisplay>` and `sig_hint: Option<SharedString>`.
8. New `ChromeView` fields: `autocomplete: Option<Autocomplete>` where
   `struct Autocomplete { matches: Vec<&'static FnSig>, highlight: usize, token_start: usize }`,
   and `sig_hint: Option<&'static str>`. Init both to `None` in `new`.
9. `recompute_autocomplete(&mut self, window, cx)` — reads the **driving** editor
   (`self.edit.origin()` → `content_input` or `edit.in_cell()`), its `value()` + `cursor()`
   (byte offset). If a cap error is visible, clear both and return (cap-error precedence).
   Else set `self.autocomplete` from `fn_edit_context` + `complete` (preserve `highlight`
   when `token_start` is unchanged, else reset to 0; capped display handled at render), and
   `self.sig_hint` from `enclosing_fn_name` + `signature`. Does **not** notify/refresh
   (callers do).
10. Call `recompute_autocomplete` in both `on_content_event`'s and `on_incell_event`'s
    `Change` arm, **before** `refresh_edit_grid_state` (so the pushed `EditState` carries the
    display state), replacing nothing else.
11. `refresh_edit_grid_state` builds `AutocompleteDisplay` (rows capped to ~10) + `sig_hint`
    from the fields and includes them in the `EditState` emit — but **only for
    `EditOrigin::InCell`** (the data row renders its own popover; the grid renders the in-cell
    one). Update the `chrome/mod.rs` enum, the `shell/window.rs` handler, and
    `GridView::set_edit_state` signature accordingly.
12. Accept algorithm `accept_autocomplete(&mut self, window, cx)` (D1.2):
    - `ac = self.autocomplete.take()`; `sig = ac.matches[ac.highlight]`.
    - Read driving editor `text` + `caret`; `insertion = format!("{}(", sig.name)`;
      `new_text = text[..ac.token_start] + insertion + text[caret..]`;
      `new_caret = ac.token_start + insertion.len()`.
    - `self.data_row.reduce(Edited { new_text })` + `apply_data_effects` (canonical text),
      then set the driving editor's value + caret: `set_value(new_text)` (suppresses Change)
      then `set_cursor_position(Position::new(0, char_col))` where `char_col =
      new_text[..new_caret].chars().count()`. Mirror to the other editor (syncing guard).
    - `self.sig_hint = Some(sig.template)`; `refresh_edit_grid_state`; `cx.notify()`.
13. Navigation/dismiss methods: `autocomplete_nav(down)`, `autocomplete_dismiss` (clears the
    list only — edit continues), `autocomplete_accept` (= `accept_autocomplete`),
    `autocomplete_accept_at(index)` (sets highlight then accepts). All `pub` (window routes
    the in-cell events here). `nav` clamps (no wrap).
14. Data-row key interception: in `handle_data_row_edit_key`, when `self.autocomplete.is_some()`
    (checked before the Tab/quick-edit logic), consume `down`/`up` (nav), `enter`/`tab`
    (accept), `escape` (dismiss) and return `true`; otherwise fall through unchanged.

### C. `freecell-app` — grid in-cell rendering + key interception (`grid/view.rs`, `grid/mod.rs`, `shell/window.rs`)

15. `GridView` new fields `incell_autocomplete: Option<AutocompleteDisplay>` +
    `incell_sig_hint: Option<SharedString>`, set via the extended `set_edit_state`; cleared
    in `set_active_sheet` alongside `incell_cap`.
16. `in_cell_overlay_elements`: below the editor (anchor `(x, y + h + 2)`, like the in-cell
    cap popover), when `incell_cap.is_none()`, render the autocomplete list card (highlighted
    row styled; each row `on_mouse_down` → `GridEvent::AutocompleteAcceptAt(i)`) and/or the
    one-line sig-hint strip (list precedence).
17. Grid `capture_key_down`: at the top, when `incell_autocomplete.is_some()`, intercept
    `down`/`up` → `AutocompleteNav`, `enter`/`tab` → `AutocompleteAccept`, `escape` →
    `AutocompleteDismiss` (each `stop_propagation`), before the existing Tab/Esc/quick-edit
    arms.
18. New `GridEvent`s: `AutocompleteNav { down: bool }`, `AutocompleteAccept`,
    `AutocompleteAcceptAt(usize)`, `AutocompleteDismiss`. Route them in `shell/window.rs` to
    the matching `ChromeView` methods (guarded on the chrome slot, like `InCellCancel`).

## Tests

### `freecell-core` unit (`functions.rs` `#[cfg(test)]`)

- `catalog_has_345_unique_nonempty_names` — count == 345, all unique, all non-empty.
- `common_names_have_real_templates` — every `rank == 0` entry's template is not the
  `NAME(…)` fallback and contains `(`.
- `complete_orders_exact_then_rank_then_alpha` — `complete("sum")` puts `SUM` first (exact);
  `complete("su")` groups `rank 0` before `rank 1`, alphabetical within a rank;
  case-insensitive (`"SU"` == `"su"`).
- `complete_empty_prefix_is_empty` and `complete_no_match_is_empty` (`"zzz"`).
- `signature_is_case_insensitive` — `signature("sumif")` == `signature("SUMIF")`.
- `fn_edit_context_truth_table` — `=su`→Some(prefix "su", token_start 1); `=SUM(A1,su`→Some
  at the 2nd arg; `=1+su`→Some; `=A1`→None (cell ref); `="su`→None (string); `su`→None
  (no `=`); `=LOG10`→Some (real name); caret-in-middle (`=su|+1`)→Some.
- `enclosing_fn_name_finds_call` — `=SUM(A1,` → "SUM"; nested `=IF(SUM(A1,` at inner →
  "SUM", after the inner closes → "IF"; outside any call (`=A1`) → None.

### `freecell-app` gpui view tests (`chrome/view.rs` `#[cfg(test)]`)

- `typing_prefix_opens_list_with_expected_order` — `begin_typed("=su")` + `recompute` →
  `autocomplete` open, `SUM`/`SUMIF`/… present, a common name highlighted first.
- `nav_moves_highlight_clamped` — down/up move + clamp at both ends.
- `accept_inserts_name_paren_and_places_caret` — accept SUM → `content_input.value()` ==
  `"SUM("`, `cursor()` == 4, `sig_hint` set, list closed.
- `accept_mid_formula_keeps_suffix` — `"=1+su"` caret 5 accept → `"=1+SUM("`, caret 7.
- `esc_closes_list_keeps_edit` — dismiss → `autocomplete` None, still `Editing`, text
  unchanged.
- `sig_hint_shows_when_caret_inside_call` — text `"=SUM("` recompute → `sig_hint` == the SUM
  template.
- `in_cell_path_opens_and_accepts` — `begin_in_cell` + push `"=su"` into the in-cell input +
  recompute (origin InCell) → list open; `accept` mutates the **in-cell** input to `"SUM("`
  and pushes an `EditState` with `autocomplete: None`.
- `non_formula_never_triggers` — `begin_typed("su")` (no `=`) + recompute → `autocomplete`
  None.
